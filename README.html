Overview

DeepTest is a production-grade automated essay scoring (AES) system built on Meta's Llama 3.1 8B language model. Essays are scored on a 1–6 scale using prompt-specific LoRA adapters trained on the ASAP dataset. Each score is accompanied by a written justification generated via the Groq API. A teacher audit portal enables human review, score overrides, and continuous alignment measurement via Quadratic Weighted Kappa (QWK).

The system is deployed on Google Cloud Platform with an NVIDIA L4 GPU, served through vLLM, and exposed via a FastAPI backend with Nginx reverse proxy and SSL.

System Architecture

The base Llama 3.1 8B model loads once into GPU memory. LoRA adapters are dynamically selected at inference time based on the prompt_id in each request. Score justifications are generated separately via the Groq API.

Core Features

FeatureDescription
Automated Essay Scoring Fine-tuned model scores essays 1–6. Essays below 200 words are automatically scored 0 and returned without inference.
Prompt-Specific Adapters Seven independent LoRA adapters, each trained for a specific essay type and its corresponding scoring rubric.
Justification Generation Each score includes a written explanation generated via Groq API to support student feedback and teacher review.
Human Audit Pipeline Teachers validate scores, apply overrides, submit comments, and flag essays for appeal through a dedicated portal.
Confidence-Based Routing Predictions below confidence threshold 0.75 are held for mandatory human review before being finalized.
Continuous QWK Evaluation Human feedback is used to compute QWK between AI and human scores, tracked continuously in the dashboard.

Scoring Rubric

Essays are scored on a 6-point scale. A score of 0 is returned automatically for submissions below the 200-word minimum, without running model inference.

ScoreDescription
6Sophisticated language control with strong argument and evidence
5Clear argument with developed reasoning and supporting detail
4Adequate writing with some development and organizational structure
3Partial development and limited clarity throughout
2Minimal structure and weak reasoning; ideas underdeveloped
1Very limited content and language control
0Essay below 200-word minimum — scoring not performed

Known Limitations

Length Bias

Longer essays may receive higher scores independent of quality. A 200-word minimum is enforced; submissions below this receive a score of 0 without inference.

Vocabulary Bias

Rare or sophisticated vocabulary can disproportionately influence predicted scores regardless of overall argument quality. This is a known artifact of training on the ASAP dataset.

Prompt 6 Underperformance

The Descriptive / Observational adapter (Prompt 6) achieves a QWK of 0.413, significantly below the system average of 0.633. This adapter is flagged for retraining. Confidence-based routing ensures these essays receive human review in the interim.

Confidence Threshold

Predictions with confidence below 0.75 are held for mandatory human review. This threshold is configurable in config/default.yaml.

Local Setup

1. Clone and create environment

Shell
# Clone the repository
git clone https://github.com/623dks/aes-platform.git
cd aes-platform

# Create and activate environment
conda create -n aes-llama-310 python=3.10
conda activate aes-llama-310

# Install dependencies
pip install vllm fastapi uvicorn asyncpg groq PyPDF2 python-docx pyyaml

2. Start PostgreSQL

Shell
docker run -d \
  --name aes_postgres \
  -e POSTGRES_USER=aes_user \
  -e POSTGRES_PASSWORD=aes_password \
  -e POSTGRES_DB=aes_db \
  -p 5432:5432 \
  postgres:15

3. Start the API server

Shell
DATABASE_URL="postgresql://aes_user:aes_password@localhost:5432/aes_db" \
GROQ_API_KEY="your_key_here" \
uvicorn backend.main:app --host 0.0.0.0 --port 8000
